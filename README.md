На Linux гораздо легче сделать бэкап базы данных.
Чтобы сделать его, используйте утилиту pg_dump. Выполните команду:
pg_dump -U postgres dostup > /opt/dostup-server/backup/backup_$(date +%Y%m%d_%H%M%S).sql

Для автоматизации бэкапа базы данных PostgreSQL на Linux можно использовать утилиту pg_dump в сочетании с планировщиком задач cron. 
Во вложении скрипт, который делает бэкап в директорию с установленным сервером.
Проверьте, что скрипт работает:
./backup_postgres.sh
Делаем скрипт исполняемым:
chmod +x /path/to/backup_postgres.sh

Настройка cron для запуска по расписанию:
1. Откройте редактор crontab:
crontab -e
2. Добавьте строку для запуска скрипта, например, каждый день в 2:00:
0 2 * * * /path/to/backup_postgres.sh >> /path/to/backup.log 2>&1
   - 0 2 * * * — задаёт выполнение в 2:00 ежедневно.
   - >> /path/to/backup.log 2>&1 — логирует вывод в файл.
